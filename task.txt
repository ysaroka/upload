Дано некоторое количество FTP и SFTP серверов с аутентификацией 
по username/password. Конфигурации которых храняться в конфиге. 
FTP - plain без SSL. Соотв. для каждого указывается абсолютный путь хранения данных. Пример:
FTP; 192.168.0.1; user1/password1; /datastore
FTP; 192.168.0.1; user2/password2; /mnt/net/
SFTP; 192.168.0.10; user2/password2; /
SFTP; 192.168.0.1; user1/password1; /sftp/
SFTP; 192.168.0.18; user2/password2; 
SFTP; 192.168.0.11; user2/password2; /myfiles

Даны описания принимаемых файлов от юзера. Расширения, соотв. максимальные размеры и для 
некоторых файлов наборы стоп фраз. При этом для одного расширения может быть указано 2 записи. Например:
.txt; max: 10000; stopwords: первая фраза,вторая фраза
.txt; max: 999999; stopwords: это еще одна фраза
.wav; max: 999999999;
.avi;

задача приложения - принять у юзера файл, валидировать по расширению/размеру/стоп фразам, организовать 
локальное хранилище файлов и залить на все FTP и SFTP сервера, которые доступны на данный момент.

Процедура заливки на сервера работает в ассинхронном режиме с отображением прогресса 
заливки в браузере клиенту после приема от него файла. По завершению выводится список залитых 
файлов в виде таблички, где col = каждый сервер, row = файл.
сервера могут добавляться/убираться в любое время, но информация о них должна выводится 
даже если в текущей конфигурации его нет.

Фреймворк - любой. Обязательно ORM. Предпочтение Symfony2
База данных - любая. Предпочтение mysql
css/js по усмотрению.
Ассинхронный процесс - любая реализация кроме cron. Предпочтение rabbit

установка приложения - github + composer
